# Stack Research

**Domain:** third-party email skill integration for opensquad
**Researched:** 2026-03-24
**Confidence:** HIGH

## Recommended Stack

### Core Technologies

| Technology | Version | Purpose | Why Recommended |
|------------|---------|---------|-----------------|
| Resend MCP Server (`resend-mcp`) | latest via `npx -y resend-mcp` | Primary integration path for email send, batch send, and contacts | Official MCP server already supports the exact operations needed for v1 and optional contact import, which minimizes custom code and maintenance |
| Node.js CLI integration in `opensquad` | Node.js 20+ | Guided setup, config persistence, and MCP registration inside the existing product | The repo is already a Node.js CLI with prompt-driven setup and file-based installation flows in `src/init.js`, `src/skills.js`, and `src/skills-cli.js` |
| SKILL.md-based bundled skill | current `opensquad` format | Ship the Resend capability through the existing catalog/install system | The codebase already distributes skills from `skills/<id>/SKILL.md`, including MCP skills such as `skills/blotato/SKILL.md` |

### Supporting Libraries

| Library | Version | Purpose | When to Use |
|---------|---------|---------|-------------|
| Resend Email API | current REST API at `https://api.resend.com` | Fallback path if MCP cannot satisfy a specific workflow | Use only when MCP lacks a required feature or when `opensquad` needs tighter control than MCP exposes |
| Resend Node SDK (`resend`) | latest supported by Resend docs | Direct Node.js fallback for email send and contact operations | Use if a later phase adds Node-based fallback implementations behind the same product UX |
| `@inquirer/*` prompt libraries already in repo | current repo version set | Interactive setup for API key, sender address, and domain guidance | Use for non-developer onboarding so users are guided instead of being told to edit env vars manually |

### Development Tools

| Tool | Purpose | Notes |
|------|---------|-------|
| `npx opensquad install <skill>` and bundled catalog | Skill installation and update path | Current CLI supports bundled skills, but MCP-specific setup UX appears incomplete for automatic onboarding |
| IDE MCP config files generated by templates | Register MCP servers for Claude/Codex/Cursor/OpenCode-style clients | Existing templates already target multiple IDEs, so Resend onboarding should integrate with those templates instead of inventing a parallel path |
| `.env` plus local preferences/state files | Persist secrets and sender defaults | Repo already uses file-based persistence; guided setup can write env/config safely without forcing manual editing |

## Installation

```bash
# Core product dependencies already exist
npm install

# Primary integration is external and intentionally reused
npx -y resend-mcp

# Optional direct fallback only if MCP proves insufficient later
npm install resend
```

## Alternatives Considered

| Recommended | Alternative | When to Use Alternative |
|-------------|-------------|-------------------------|
| Resend MCP Server | Direct Resend API calls from `opensquad` | Use direct API only if MCP cannot cover a verified product need such as a missing workflow or control point |
| Bundled MCP skill plus guided installer | Prompt-only “how to send email” instructions | Only acceptable for internal experiments; not acceptable for the non-technical UX goal |
| Automatic configuration persistence | Manual env editing by the user | Only acceptable for advanced users or temporary debugging workflows |

## What NOT to Use

| Avoid | Why | Use Instead |
|-------|-----|-------------|
| Building a full custom email transport first | Reinvents already-available official tooling and expands maintenance surface too early | Start with `resend-mcp` |
| Forcing users to hand-edit `RESEND_API_KEY` before install | Creates friction and conflicts with the project goal of non-developer usability | Use guided setup that collects and writes config automatically |
| Expanding v1 to all Resend capabilities | Increases scope before the core email path is proven inside squads | Focus on single send, batch send, and possibly contact import |

## Stack Patterns by Variant

**If v1 remains MCP-first:**
- Use a bundled `mcp` skill similar to `skills/blotato/SKILL.md`
- Because Resend already documents MCP setup for Codex and other clients, including stdio and HTTP modes

**If a fallback is needed later:**
- Add a `hybrid` or script-backed layer that calls the Resend Node SDK
- Because the product may eventually need product-specific automation beyond the MCP abstraction

## Version Compatibility

| Package A | Compatible With | Notes |
|-----------|-----------------|-------|
| `opensquad` CLI on Node.js 20+ | `npx -y resend-mcp` | Compatible at process boundary because the MCP server runs as a separate command |
| Resend REST API | Bearer API keys and mandatory `User-Agent` header | Direct API fallback must send `User-Agent` or requests can be rejected with `403` |
| Batch send endpoint | Resend Email API | Supports up to 100 emails per API call, but does not yet support `attachments` or `scheduled_at` in batch mode |
